01 O que é este trabalho?

Objetivo central

Servidor Local de Arquivos para o IPIKK

O trabalho consiste na criação e configuração de um servidor de arquivos dentro do Instituto Politécnico Industrial do Kilamba Kiaxi (IPIKK), transformando um computador comum em um repositório central onde todos os documentos administrativos das coordenações ficam guardados de forma organizada, segura e acessível apenas a utilizadores autorizados.

02 Problema e Solução

⚠️ Antes — O Problema

  • Arquivos guardados em pen drives e discos externos
  • Partilha informal via WhatsApp e e-mail
  • Risco elevado de perda de dados
  • Sem controlo de quem acede aos documentos
  • Duplicação e desorganização de ficheiros
  • Sem cópias de segurança (backup)

✅ Depois — A Solução

  • Repositório central único para todas as coordenações
  • Acesso controlado por utilizador e grupo
  • Permissões NTFS definidas por departamento
  • Partilha segura via protocolo SMB na rede local
  • Estrutura de pastas organizada por coordenação
  • Base para futuras cópias de segurança automáticas

04 Conceitos Fundamentais

Modelo de Rede

Arquitetura Cliente-Servidor

O servidor centraliza os recursos e responde às solicitações. Os clientes (computadores das coordenações) fazem pedidos de acesso. Esta divisão garante organização, controlo e atendimento simultâneo de múltiplos utilizadores sem conflitos.

Sistema de Ficheiros

Permissões NTFS — Os 4 Níveis

  • Leitura — só visualizar o conteúdo da pasta
  • Escrita — criar e modificar ficheiros
  • Execução — executar programas armazenados
  • Controlo Total — gerir permissões e configurações (apenas administrador)
Protocolo de Partilha

SMB vs FTP vs SFTP

  • SMB — ideal para redes LAN Windows; integração nativa com o Windows Server; porta 445; escolhido neste projeto
  • FTP — transferência entre sistemas; sem criptografia; portas TCP 20 e 21
  • SFTP — versão segura do FTP; opera sobre SSH; porta TCP 22; recomendado para acesso externo
Rede Local

Por que usar IP Fixo no servidor?

Com IP dinâmico (DHCP), o endereço do servidor pode mudar após uma reinicialização, quebrando todas as ligações mapeadas nos computadores clientes. O IP fixo garante que as coordenações encontram sempre o servidor no mesmo endereço, assegurando continuidade do serviço.

05 Etapas da Implementação

Fase 1 — Análise

Levantamento bibliográfico e diagnóstico da infraestrutura existente no IPIKK; aplicação de inquéritos a coordenadores, professores e técnicos de TI.

Fase 2 — Instalação

Instalação do Windows Server 2019 Standard (Desktop Experience) via pen drive bootável; criação da partição primária no HDD de 500 GB.

Fase 3 — Configuração de Rede

Atribuição do endereço IP estático; confirmação de conectividade via comando ping nos terminais clientes.

Fase 4 — Utilizadores e Grupos

Criação de 7 grupos (DP, EIE, FC, GSI, INF, OCC, TM) com 5 utilizadores cada, totalizando 35 contas com senhas complexas.

Fase 5 — Pastas e Permissões

Criação das pastas por coordenação; definição de permissões NTFS (Leitura e Escrita para cada grupo; Controlo Total para o administrador); pasta SETUP partilhada por todos.

Fase 6 — Testes

7 testes realizados: conectividade, acesso às pastas, permissões de leitura/escrita, bloqueio de acesso não autorizado, acesso simultâneo e mapeamento de unidade de rede — todos aprovados.

06 Estrutura de Acesso às Pastas

Modelo híbrido de permissões

Dois Níveis de Acesso

Acesso Comum: todos os 7 grupos podem aceder à pasta SETUP, que funciona como repositório partilhado de ferramentas e recursos comuns a toda a instituição.

Acesso Restrito: cada grupo acede exclusivamente à sua própria pasta (ex.: Grupo GSI → Pasta GSI), garantindo confidencialidade dos dados de cada coordenação.

Parte 1 — Conceitos Fundamentais
Q01 O que é um servidor e qual a sua função principal? ▼
✔ Resposta

Um servidor é um computador ou programa que fornece serviços, dados ou recursos a outros computadores — chamados clientes — através de uma rede. Permanece em estado de escuta contínua, respondendo às requisições recebidas. Distingue-se de um computador pessoal pelo seu funcionamento contínuo, alta disponibilidade e capacidade de atender múltiplos utilizadores em simultâneo.

Q02 O que é o modelo cliente-servidor e como se diferencia do modelo P2P? ▼
✔ Resposta

No modelo cliente-servidor, as funções são claramente separadas: o cliente inicia a comunicação e solicita serviços; o servidor processa e responde. Já no modelo P2P (Peer-to-Peer), todos os nós têm funções equivalentes, sem um nó central especializado. O modelo cliente-servidor oferece maior organização, controlo de acessos e centralização de recursos.

Q03 Quais são os três componentes fundamentais de um servidor? ▼
✔ Resposta
  • Hardware — componentes físicos: CPU, RAM, disco de armazenamento e placa de rede
  • Sistema Operativo de Servidor (SOS) — gere os recursos e disponibiliza serviços; ex.: Windows Server 2019
  • Serviços de rede — definem as funcionalidades oferecidas aos clientes; ex.: partilha de ficheiros via SMB
Q04 O que é uma LAN e por que é adequada para este projeto? ▼
✔ Resposta

LAN (Local Area Network) é uma rede de computadores interligados numa área geográfica limitada (escola, escritório, edifício). É adequada para este projeto porque todas as coordenações do IPIKK estão no mesmo complexo, permitindo alta taxa de transferência, baixa latência e maior controlo administrativo, sem depender da Internet pública.

Q05 Pode-se usar um computador comum como servidor? Quais são as limitações? ▼
✔ Resposta

Sim. Computadores comuns (gabinetes adaptados) podem funcionar como servidores em ambientes de pequeno porte, como o IPIKK. Contudo, apresentam limitações: menor robustez, maior risco de sobreaquecimento, dificuldade de expansão e menor tolerância a falhas. Para mitigar estes riscos, recomenda-se UPS, backups periódicos, monitorização de hardware e manutenção preventiva.

Parte 2 — Problema, Solução e Justificação
Q06 Qual era o problema principal de gestão de dados no IPIKK antes do projeto? ▼
✔ Resposta

As coordenações armazenavam ficheiros de forma descentralizada em computadores individuais, pen drives, discos externos e aplicações de mensagens (WhatsApp, e-mail), sem qualquer sistema de backup. Isto gerava: risco elevado de perda de dados, duplicação de ficheiros, ausência de controlo de acessos e dificuldade na partilha segura de documentos.

Q07 Qual é a hipótese central do trabalho e foi confirmada? ▼
✔ Resposta

A hipótese central era que a implementação de um servidor de arquivos melhoraria a organização dos dados, reduziria a perda de informações e aumentaria a segurança através de políticas de permissões. Sim, foi confirmada: todos os 7 testes de funcionamento foram aprovados, demonstrando que a solução é viável e eficaz no contexto do IPIKK.

Q08 Por que o projeto se limita a uma rede local (LAN) e não inclui soluções em nuvem? ▼
✔ Resposta

A delimitação do estudo é explícita: o projeto cobre apenas a implementação interna em rede local, não abrangendo serviços externos em nuvem ou sistemas de gestão académica. Esta escolha deve-se à necessidade de controlo interno dos dados, redução de custos operacionais e funcionamento independente da conectividade com a Internet — fatores críticos numa instituição de ensino como o IPIKK.

Parte 3 — Tecnologias e Protocolos
Q09 Por que o Windows Server 2019 foi escolhido em vez do Linux? ▼
✔ Resposta

O Windows Server 2019 foi selecionado pela sua compatibilidade com o hardware existente, interface gráfica intuitiva (facilita a administração), integração nativa com o protocolo SMB e permissões NTFS, suporte a Active Directory, e familiaridade da equipa técnica do IPIKK com o ambiente Windows. Em contextos de baixo custo com hardware convencional, também se mostra tecnicamente viável.

Q10 Explique o protocolo SMB: o que é, como funciona e por que foi escolhido? ▼
✔ Resposta

O SMB (Server Message Block) é um protocolo de partilha de ficheiros amplamente utilizado em redes Windows. Opera sobre TCP/IP, normalmente pela porta 445. Permite partilha direta de pastas e impressoras, integra-se ao sistema de autenticação do Windows Server, e oferece controlo granular de permissões, auditoria de acessos e mapeamento de unidades de rede. Foi escolhido por ser a solução mais adequada para o ambiente LAN interno do IPIKK.

Q11 Qual a diferença entre FTP e SFTP? Quando se usaria cada um? ▼
✔ Resposta
  • FTP — transferência de ficheiros sem criptografia; portas TCP 20 e 21; adequado para redes internas protegidas, mas expõe credenciais em redes não seguras
  • SFTP — opera sobre o protocolo SSH (porta TCP 22); garante criptografia, autenticação segura e proteção contra intercepção; indicado para acesso externo ou transferência de dados sensíveis

No IPIKK, o SMB foi preferido para partilha interna contínua; FTP/SFTP poderiam ser usados para backups remotos futuros.

Q12 O que é o NTFS e quais permissões foram aplicadas no projeto? ▼
✔ Resposta

NTFS (New Technology File System) é o sistema de ficheiros do Windows que permite definir regras granulares de acesso. No projeto:

  • Cada coordenação (grupo) recebeu Leitura e Escrita sobre a sua pasta
  • O administrador do sistema tem Controlo Total
  • Todos os grupos têm acesso à pasta SETUP (recurso comum)
  • Grupos não têm acesso às pastas de outras coordenações
Q13 Por que é essencial atribuir um endereço IP fixo ao servidor? ▼
✔ Resposta

Com IP dinâmico (DHCP), o endereço pode mudar após reinicializações ou renovação de concessão, causando falhas nas ligações mapeadas pelos clientes. O IP fixo garante que os computadores das coordenações encontram o servidor sempre no mesmo endereço, assegurando continuidade do serviço, facilidade de implementação de regras de firewall, monitorização e documentação da infraestrutura.

Parte 4 — Implementação e Hardware

Parte 5 — Segurança e Controlo de Acesso
Q17 O que é autenticação e como é gerida no Windows Server 2019? ▼
✔ Resposta

Autenticação é o processo de verificar a identidade do utilizador antes de conceder acesso ao sistema. No Windows Server 2019, é gerida através do Active Directory (AD), que centraliza contas de utilizadores, grupos e permissões. Políticas de senhas complexas e renovação periódica aumentam significativamente a segurança e previnem acessos não autorizados.

Q18 Quais são as múltiplas camadas de segurança recomendadas para um servidor de arquivos? ▼
✔ Resposta
  • Segurança física — proteger o hardware contra acesso físico não autorizado e falhas ambientais
  • Segurança lógica — autenticação, criptografia e políticas de permissões NTFS
  • Monitorização — registo de acessos e alterações para auditoria
  • Backups periódicos — cópias externas ou em nuvem para recuperação em caso de falha
Q19 O que é o modelo híbrido de acesso implementado no projeto? ▼
✔ Resposta

O modelo combina dois níveis: Acesso Comum — todos os 7 grupos acedem à pasta SETUP (repositório de recursos partilhados); e Acesso Restrito — cada grupo tem acesso exclusivo à sua própria pasta, sem conseguir ver as pastas dos outros grupos. Este modelo equilibra colaboração e confidencialidade.

Parte 8 — Recomendações e Futuro
Q24 Quais são as principais recomendações para melhorar o sistema no futuro? ▼
✔ Resposta
  • RAID 1 — adicionar segundo disco para espelhamento e redundância de dados
  • Backups automáticos diários — usar Windows Server Backup para cópias em disco externo ou nuvem
  • Active Directory Domain Services (AD DS) — migrar para controlador de domínio com políticas de grupo (GPO)
  • Hardware dedicado — adquirir servidor com SSD/NAS, mais RAM e fonte redundante (UPS)
  • Monitorização contínua — ativar alertas automáticos de desempenho e tentativas de acesso
  • Expansão para DHCP e servidor de impressão — centralizar mais serviços de TIC
  • Formação dos utilizadores — sessões de boas práticas para as coordenações
Q25 O que é o RAID e qual modalidade foi recomendada? Por que é útil? ▼
✔ Resposta

RAID (Redundant Array of Independent Disks) é uma tecnologia que combina múltiplos discos para melhorar desempenho ou redundância. Foi recomendado o RAID 1 (espelhamento): os dados são escritos simultaneamente em dois discos, de modo que se um falhar, o outro mantém a informação intacta, garantindo continuidade operacional sem perda de dados.

Q26 O que é o Active Directory Domain Services (AD DS) e qual vantagem traz sobre a gestão local? ▼
✔ Resposta

O AD DS transforma o servidor num Controlador de Domínio, permitindo gestão centralizada e avançada de todos os utilizadores e computadores da rede. Vantagens: autenticação de domínio (utilizador pode entrar em qualquer computador da escola com as suas credenciais), políticas de grupo (GPO) para definir configurações automáticas, e gestão muito mais eficiente do que a criação de utilizadores locais usada atualmente.

Parte 9 — Perguntas de Aprofundamento
Q27 Qual é a diferença entre permissões de partilha e permissões NTFS? Qual prevalece? ▼
✔ Resposta

As permissões de partilha controlam o acesso à pasta quando acedida pela rede; as permissões NTFS controlam o acesso ao nível do sistema de ficheiros (local e remoto). Quando ambas estão configuradas, o acesso efetivo é determinado pela combinação mais restritiva das duas. No projeto, as permissões foram configuradas de forma coerente para que as NTFS fossem o controlo principal.

Q28 O que é um UPS e por que é importante para a continuidade do servidor? ▼
✔ Resposta

UPS (Uninterruptible Power Supply) é um sistema de alimentação ininterrupta — uma bateria que entra em funcionamento automaticamente quando há corte ou instabilidade de energia elétrica. É crucial para servidores porque uma queda súbita de energia pode corromper dados em escrita, danificar o sistema de ficheiros ou provocar perda irreversível de informação.

Q29 Como é feito o mapeamento de unidade de rede nos computadores clientes? ▼
✔ Resposta

O mapeamento é feito através do caminho de rede \\IP_do_servidor (ex.: \\192.168.X.X). No Windows, acede-se via Explorador de Ficheiros → "Mapear unidade de rede", indicando o caminho da pasta partilhada. Após autenticação com as credenciais do utilizador, a pasta aparece como uma unidade de disco (ex.: Z:) no computador cliente, facilitando o acesso direto sem precisar de digitar o caminho completo.
